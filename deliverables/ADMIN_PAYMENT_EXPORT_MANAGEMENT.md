# 관리자 결제관리 및 엑셀 다운로드 산출물

## 요구내용
- 관리자페이지에 `결제 관리` 메뉴를 구성한다.
- 결제 관리는 그리드 형태로 `결제번호, 보호자, 대상자, 구분, 결제수단, 결제금액, 결제일`을 조회한다.
- 관리자페이지의 각 그리드에는 엑셀 다운로드 기능을 제공한다.

## 반영내용
- 기존 `결제 관리` 메뉴에 결제내역 그리드를 추가했다.
- 결제내역은 최대 300건을 최신순으로 조회한다.
- 결제내역 통합 검색은 결제번호, 보호자명, 보호자 연락처, 보호자 이메일, 대상자명, 결제 구분, 결제수단을 대상으로 한다.
- 결제 구분 필터는 `전체`, `이용권`, `상품`, `광고`로 구성했다.
- `이용권 옵션 가격` 설정 화면은 결제내역 아래에 유지했다.
- 관리자 그리드의 다운로드는 브라우저에서 UTF-8 BOM 포함 CSV를 생성한다.
- CSV 파일은 Excel에서 바로 열 수 있으며, 필터 적용 후 화면에 조회된 결과 기준으로 내려받는다.

## 결제내역 데이터 기준
- 상품/이용권 결제는 `product_orders`의 결제 완료 상태를 기준으로 조회한다.
- 포함 상태는 `paid`, `paid_waiting_activation`, `activated`이다.
- 상품 단독 구매는 `상품`, 이용권 구매는 `이용권`으로 표시한다.
- 광고는 현재 `subject_ads`의 광고 금액 이력을 함께 표시한다.
- 광고 결제수단은 실제 Meta/Toss 광고 결제 API 연동 전 단계이므로 `광고 결제 준비`로 표시한다.

## 엑셀 다운로드 적용 범위
- 결제 관리: 결제내역
- 보호자 관리: 보호자 목록
- 관리대상자 관리: 관리대상자 목록
- QR 관리: 생성된 QR 목록
- 관리자 관리: 가입된 사용자 목록
- 상품 관리: 상품 목록
- 주문/배송 관리: 주문 목록
- 광고 관리: 광고 목록
- 실종신고 관리: 실종신고 목록
- 위치공유 관리: 위치공유 목록
- 고객문의: 고객문의 목록

## 주요 파일
- `app/admin/page.js`
  - 결제내역 그리드, 검색/구분 필터, CSV 다운로드 데이터 매핑
  - 관리자 각 목록의 다운로드 버튼 연결
- `app/admin/export-button.js`
  - 클라이언트에서 CSV를 생성하고 다운로드하는 공통 버튼
- `lib/db.js`
  - `getAdminPaymentsData()` 결제내역 조회 함수
- `app/globals.css`
  - 결제내역 그리드, 다운로드 버튼, 제목 액션 영역 스타일

## 운영 메모
- 현재 다운로드 파일은 `.csv` 확장자이지만 Excel에서 바로 열 수 있는 UTF-8 CSV이다.
- `.xlsx`가 반드시 필요해지면 서버 또는 클라이언트에 별도 스프레드시트 라이브러리를 추가하는 방식으로 확장 가능하다.
- 광고 결제내역은 광고 API와 실제 광고 결제 흐름을 붙인 뒤 결제수단/결제상태 컬럼을 실제 결제 데이터로 대체하는 것이 좋다.

## 이미지 생성 프롬프트
> REAL_QR_FIND 관리자 콘솔 화면. 왼쪽 세로 메뉴와 숨김 토글이 있고, 중앙에는 결제 관리 그리드가 보인다. 상단에는 통합 검색, 구분 필터, 조회, 초기화, 엑셀 다운로드 버튼이 있으며, 그리드에는 결제번호, 보호자, 대상자, 구분, 결제수단, 결제금액, 결제일 컬럼이 정돈되어 있다. 공공서비스 스타일의 흰색 배경, 연한 파란색 테이블 헤더, 보라색 포인트 컬러, 업무용 SaaS 대시보드 느낌.
