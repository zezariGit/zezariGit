# 관리자 세로 메뉴 구현 산출물

## 구현 목적
- 관리자 페이지 상단의 가로 탭을 왼쪽 세로 메뉴로 변경한다.
- 관리자가 필요할 때 메뉴를 숨겨 넓은 작업 영역을 사용할 수 있게 한다.
- 모바일에서는 화면 폭을 침범하지 않도록 메뉴와 본문을 한 열로 배치한다.

## 구현 내용
- `app/admin/admin-workspace.js`
  - 관리자 메뉴와 콘텐츠 레이아웃을 담당하는 클라이언트 컴포넌트
  - 메뉴 숨기기/펼치기 버튼 제공
  - `localStorage`에 접힘 상태 저장
  - 현재 선택한 관리 메뉴 강조
  - 메뉴 항목마다 전용 선형 아이콘 표시
  - 하단에 `사용자 화면`, `로그아웃` 빠른 작업 배치
- `app/admin/page.js`
  - 기존 관리 기능을 `AdminWorkspace` 안에 배치
  - 각 메뉴 URL과 서버 데이터 조회 방식은 기존과 동일하게 유지
  - 본문 헤더의 사용자 화면/로그아웃 버튼은 사이드바로 이동
- `app/auth-actions.js`
  - `LogoutButton`에 `className`, `children` 옵션을 추가해 기존 사용처는 유지하면서 관리자 사이드바 전용 표시 가능
- `app/globals.css`
  - 데스크톱: 198px 세로 메뉴와 본문 2열 구성
  - 접힘 상태: 메뉴 영역을 56px로 축소하고 메뉴 항목 숨김
  - 860px 이하: 메뉴와 본문을 한 열로 배치
  - 키보드 포커스, 호버, 모션 감소 설정 반영
  - 첨부 참고안과 유사한 진한 네이비 배경, 흰색 메뉴 텍스트, 보라색 활성 메뉴 스타일 적용

2. 보호자 관리
3. 대상자 관리
4. QR 관리
5. 주문 관리
6. 결제 관리
7. 상품 관리
8. 광고 관리
9. 실종신고 관리
10. 위치공유 관리
11. 고객 문의
12. 관리자 관리

## 2026-06-28 스타일 개선
- 첨부 참고안처럼 왼쪽 메뉴를 진한 네이비 운영 메뉴 스타일로 변경했다.
- 상단 브랜드를 `제자리 관리자`로 표시하고 방패 아이콘을 추가했다.
- 각 메뉴에 기능을 암시하는 선형 아이콘을 추가했다.
- 선택된 메뉴는 보라색 배경과 옅은 테두리로 강조한다.
- 기존 메뉴 숨김/펼치기 기능과 접힘 상태 저장 방식은 유지했다.
- `사용자 화면`과 `로그아웃` 버튼을 왼쪽 관리메뉴 하단으로 이동했다.

- Next.js 프로덕션 빌드 성공
- 데스크톱 펼침 메뉴 폭: 198px
- 데스크톱 접힘 메뉴 폭: 56px
- 새로고침 후 접힘 상태 유지 확인
- 활성 메뉴 강조 확인
- 모바일에서 메뉴 아래에 본문 배치 확인
- 데스크톱과 모바일 모두 가로 스크롤 없음
- 운영 DB와 분리된 임시 SQLite DB에서 관리자 로그인 및 화면 검증
- 테스트 DB와 화면 캡처 파일 삭제 완료
