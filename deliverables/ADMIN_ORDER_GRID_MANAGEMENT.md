# REAL_QR_FIND 관리자 주문/배송 그리드 개편 산출물

## 목적
- 주문별 대형 카드 반복을 제거하고 여러 주문을 빠르게 비교할 수 있는 관리 화면을 제공한다.
- 목록에서 주문을 선택한 뒤 한 곳에서 수령인, 배송지, 택배사, 송장번호와 관리자 메모를 조회·수정한다.

## 관리자 경로
- `/admin?section=orders`
- 선택 주문: `/admin?section=orders&order={주문ID}`

## 화면 구성
### 2026-06-29 운영형 주문관리 화면 보강
- 첨부 시안 기준으로 상단 주문 현황 카드를 6개로 확장했다.
  - 전체 주문
  - 결제 완료
  - 배송중
  - 배송완료
  - 취소
  - 환불
- 검색 패널을 주문관리 전용 영역으로 분리했다.
  - 검색어: 주문번호, 대상자명, 보호자명, 연락처, 송장번호
  - 상품명
  - 주문상태
  - 배송상태
  - 주문기간 시작/종료
- 주문 그리드를 운영형 고밀도 표로 확장했다.
  - 선택
  - 주문번호
  - 주문일시
  - 대상자명/보호자명
  - 연락처
  - 상품명(옵션)
  - 수량
  - 상품금액
  - 결제금액
  - 결제방법
  - 주문상태
  - 배송상태
  - 관리
- 오른쪽 상세카드는 주문 기본 정보, 대상자 정보, 주문 상품, 결제정보, 배송정보, 관리자 배송정보 입력 순서로 정리했다.
- 상세카드 하단의 관리자 배송정보 입력에서 택배사와 송장번호를 저장한다.
- 사용자 마이페이지의 `결제 및 이용권 현황` 주문내역에 배송조회 정보 블록을 추가해 구매자가 택배사와 송장번호를 확인할 수 있게 했다.

### 주문 현황
- 전체 주문, 결제 완료, 배송 준비, 배송 중, 배송 완료 건수를 표시한다.

### 왼쪽 주문 그리드
- 주문번호
- 주문자와 관리대상자
- 상품
- 결제금액
- 결제상태
- 배송상태
- 주문일
- 행 전체를 선택하면 오른쪽 상세정보가 해당 주문으로 변경된다.

### 오른쪽 주문 상세
- 상품, 수량, 관리대상자, 구매유형, 결제금액, 결제수단, 결제일
- 보호자 이름·연락처·이메일
- 수령인 이름·연락처
- 기본주소와 상세주소
- 배송상태, 택배사, 송장번호, 관리자 메모
- 발송일과 배송완료일

## 조회와 저장 규칙
- 검색 및 결제·배송 필터를 사용해 최대 200건을 조회한다.
- 선택한 주문 ID를 URL에 유지하므로 새로고침과 저장 후에도 같은 주문 상세가 열린다.
- 결제 완료 주문만 배송 준비, 배송 중, 배송 완료로 변경할 수 있다.
- 배송 중 또는 배송 완료 저장 시 택배사와 송장번호가 필수다.
- 관리자 메모는 여러 줄로 입력할 수 있다.
- 저장 성공·실패 결과는 기존 전역 하단 메시지로 안내한다.

## 데이터 변경
- 신규 테이블이나 컬럼은 없다.
- `getAdminOrdersData`가 필터 결과 안에서 선택 주문을 함께 반환하도록 확장됐다.
- 기존 `setProductOrderFulfillment` 검증과 알림 로직을 그대로 사용한다.

## 반응형 기준
- 넓은 화면에서는 왼쪽 그리드와 오른쪽 상세를 2열로 표시한다.
- 1200px 이하에서는 목록과 상세를 위아래 1열로 표시한다.
- 좁은 화면의 주문 그리드는 내부 가로 스크롤을 사용하며 페이지 자체에는 가로 넘침이 생기지 않는다.

## 검증 결과
- Next.js 프로덕션 빌드 성공
- 임시 DB 주문 3건 그리드 조회 및 7개 열 확인
- 주문 행 선택 후 URL·상세정보 변경 확인
- 기본주소와 상세주소 표시 확인
- 배송상태, 택배사, 송장번호, 관리자 메모 저장 후 DB 값 확인
- 결제 대기 주문의 상태 선택지를 결제 확인 전·배송 취소로 제한하는지 확인
- 배송 중 필터 결과 확인
- 1440px 데스크톱 2열 및 390px 모바일 1열 확인
- 데스크톱·모바일 페이지 가로 넘침 없음 확인

## 발표자료용 이미지 생성 프롬프트
> 한국 공공서비스형 웹 관리자 화면의 주문/배송 관리 대시보드. 흰색 배경과 절제된 파란색 포인트, 왼쪽에는 주문번호·주문자·상품·금액·결제상태·배송상태·주문일이 표시된 데이터 그리드, 선택된 행은 연한 파란색으로 강조. 오른쪽에는 선택 주문의 상품정보, 수령인과 상세주소, 배송상태, 택배사, 송장번호, 관리자 메모 입력폼이 세로로 정돈된 상세 패널. 실제 운영 쇼핑몰 백오피스처럼 밀도 있고 명확한 16:9 프레젠테이션 이미지, 한글 UI, 장식 최소화, 높은 가독성.
