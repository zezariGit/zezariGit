# 관리자 대시보드 최근현황 구현 산출물

## 구현 목적
- 관리자 대시보드 지표 카드 아래에서 최근 운영 변화를 빠르게 확인한다.
- 각 항목의 최신 4건을 보여주고 더보기 링크로 관련 관리 화면에 이동한다.

## 최근현황 카드
| 카드 | 표시 데이터 | 정렬 기준 | 더보기 연결 |
| --- | --- | --- | --- |
| 신규가입자 | 이름, 가입일시 | 회원 생성일시 최신순 | 보호자 관리 |
| 신규주문 | 주문번호, 주문일시 | 주문 생성일시 최신순 | 주문/배송 |
| QR 활성화 요청 | 관리대상 이름, 요청일시 | 결제일시 우선, QR 갱신일시 보완 | QR 관리 |
| 실종신고 현황 | 관리대상 이름, 신고일자 | `찾는중` 상태 갱신일 최신순 | 광고 관리 |
| 고객문의 | 문의 제목, 작성일시 | 문의 생성일시 최신순 | 고객문의 관리 |

## 고객문의 준비
- DB 스키마 버전을 4로 올렸다.
- `customer_inquiries` 테이블과 생성일시 인덱스를 추가했다.
- 관리자 메뉴에 고객문의 항목을 추가했다.
- 고객문의 목록은 제목, 작성자, 상태, 작성일시를 최근 100건까지 조회한다.
- 아직 사용자 문의 접수 기능이 없으므로 운영 데이터는 0건으로 표시될 수 있다.

## 화면 구성
- 데스크톱: 최근현황 카드 5열
- 중간 화면: 2열
- 모바일: 1열
- 각 카드 최소 높이를 고정하고 더보기 링크를 카드 하단에 정렬
- 긴 주문번호와 제목은 카드 폭 안에서 자동 줄바꿈
- 날짜·시간은 한국시간 기준 `YYYY.MM.DD HH:mm` 형식

## 검증 결과
- Next.js 프로덕션 빌드 성공
- 격리 SQLite DB에 각 항목 5건을 넣고 최신 4건만 노출되는지 확인
- 각 카드에서 5번째, 4번째, 3번째, 2번째 데이터가 표시되고 가장 오래된 1번째 데이터 제외 확인
- 보호자, 주문, QR, 광고, 고객문의 더보기 경로 모두 HTTP 200 확인
- 고객문의 관리 화면에서 테스트 문의 5건 조회 확인
- DB 스키마 버전 4 확인
- 데스크톱 5열과 모바일 1열, 가로 넘침 없음 확인
- 테스트 DB, 계정, 데이터, 캡처 파일 삭제 완료

## 발표자료용 이미지 생성 프롬프트
> 한국 공공서비스 스타일의 관리자 대시보드 하단 최근현황 영역. 최근현황 제목 아래 신규가입자, 신규주문, QR 활성화 요청, 실종신고 현황, 고객문의 카드 5개가 가로로 정렬되어 있다. 각 카드에는 최신 4건의 이름 또는 제목과 날짜·시간이 간결하게 표시되고 카드 오른쪽 아래에는 파란색 더보기 링크가 있다. 흰색 카드, 옅은 회색 배경, 얇은 경계선, 절제된 파란색 강조, 실제 운영 관리도구처럼 차분하고 읽기 쉬운 한국어 UI, 16:9 발표 슬라이드용.
