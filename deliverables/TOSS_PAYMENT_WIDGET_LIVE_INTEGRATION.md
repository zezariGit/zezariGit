# REAL_QR_FIND 토스 결제위젯 라이브 연동 산출물

## 목적
- 새로 발급한 토스페이먼츠 결제위젯 라이브 키를 상품 단독구매 결제에 적용한다.
- 기존 구독 자동결제의 빌링키 연동을 유지하면서 상품 결제와 구독 결제의 키와 승인 흐름을 분리한다.

## 키 구성
- `TOSS_CLIENT_KEY`, `TOSS_SECRET_KEY`: 기존 구독 빌링 인증과 정기결제에 사용
- `TOSS_WIDGET_CLIENT_KEY`, `TOSS_WIDGET_SECRET_KEY`: 상품 단독구매 결제위젯과 결제 승인에 사용
- 클라이언트 키만 브라우저에 전달한다.
- 시크릿 키는 서버의 결제 승인 API에서만 사용하며 응답·로그·Git에 포함하지 않는다.

## 상품 결제 흐름
1. 사용자가 구독중인 상태에서 상품 단독구매를 선택한다.
2. 주문정보 화면 진입 시 `/api/payments/toss/widget/config`가 인증된 사용자에게 위젯 클라이언트 설정과 해시된 고객키를 반환한다.
3. 브라우저에서 `tossPayments.widgets()`를 생성한다.
4. 서버에서 계산된 상품금액과 같은 금액으로 `setAmount()`를 호출한다.
5. 토스 SDK가 결제수단과 필수약관 UI를 직접 렌더링한다.
6. 결제하기를 누르면 `/api/payments/toss/product/prepare`가 최신 배송지·상세주소와 주문금액으로 결제대기 주문을 생성한다.
7. 위젯의 `requestPayment()`가 토스 결제 인증을 시작한다.
8. 성공 URL에서 로그인 사용자, 내부 주문 ID, 토스 주문번호, 서버 저장금액을 다시 검증한다.
9. 서버가 `TOSS_WIDGET_SECRET_KEY`로 `/v1/payments/confirm`을 호출한다.
10. 승인 결과가 `DONE`이고 주문번호·금액이 일치할 때만 주문을 결제완료·배송준비로 변경한다.

## 구독 결제 보호
- 구독은 `payment.requestBillingAuth()`로 빌링키 인증을 시작한다.
- 기존 `TOSS_CLIENT_KEY`, `TOSS_SECRET_KEY`를 유지하므로 새 결제위젯 키 적용이 구독 흐름을 덮어쓰지 않는다.
- 상품 단독구매와 구독 결제의 시크릿 키가 서로 섞이지 않는다.

## 보안 및 데이터 규칙
- 고객키는 사용자 식별자를 SHA-256으로 해시한 고정 형식이며 원래 식별자를 브라우저에 노출하지 않는다.
- 브라우저가 전달한 결제금액만 믿지 않고 DB에 저장된 주문금액과 승인 결과를 비교한다.
- 카드번호, CVC, 계좌정보는 프로젝트 서버와 DB에 저장하지 않는다.
- 상품 결제 준비 단계에서는 결제수단을 `WIDGET`으로 기록하고 승인 후 토스 응답의 실제 결제수단으로 갱신한다.
- 성공 URL 재호출은 이미 결제완료된 주문을 재승인하지 않는다.

## 환경변수 반영
- 로컬 `.env.local`의 결제위젯 키를 별도 변수명으로 활성화했다.
- Vercel Production과 Development 환경에 두 결제위젯 키를 암호화 등록했다.
- 환경변수 값은 본 산출물과 로그에 기록하지 않는다.

## 검증 결과
- Next.js 프로덕션 빌드 성공
- `/api/payments/toss/widget/config` 빌드 및 인증 응답 확인
- 라이브 결제위젯 클라이언트 키 유형 인식 확인
- 430px 모바일 화면에서 토스 결제수단·필수약관 iframe 렌더링 확인
- 상품 단독구매 결제하기 버튼 활성화 확인
- 페이지 가로 넘침 없음 확인
- 상품 주문 준비 API가 `WIDGET`, 결제대기, 5,000원, 상세주소를 저장하는지 확인
- 테스트 주문과 임시 DB 삭제 완료
- 실제 카드·계좌 입력 및 최종 승인 거래는 자동 테스트에서 실행하지 않음

## 발표자료용 이미지 생성 프롬프트
> 한국 모바일 QR 안심서비스의 실제 상품 결제 화면. 흰색과 절제된 파란색 기반의 공공서비스 디자인, 상단에 구매 상품과 배송지·상세주소, 주문 요약이 있고 하단에는 토스페이먼츠 결제위젯이 렌더링되어 신용·체크카드와 간편결제, 필수 약관을 선택할 수 있다. 최하단에 파란색 결제하기 버튼. 보안성과 실제 운영 결제를 강조하는 9:16 스마트폰 UI 프레젠테이션 이미지, 한글 텍스트, 높은 가독성, 장식 최소화.
