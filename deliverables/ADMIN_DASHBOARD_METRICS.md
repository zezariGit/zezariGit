# 관리자 운영 대시보드 구현 산출물

## 구현 목적
- 관리자 페이지 첫 화면에서 서비스 핵심 운영 수치를 한눈에 확인한다.
- 회원·관리대상·QR·실종·광고 현황과 월별 상품·구독 매출을 같은 화면에서 비교한다.
- 선택한 월에 따라 매출 지표를 다시 조회할 수 있게 한다.

## 제공 지표
| 지표 | 집계 기준 |
| --- | --- |
| 전체 회원 | `guardians` 전체 행 수 |
| 관리대상자 | `subjects` 전체 행 수 |
| 활성 QR | 관리자 활성 상태이고 사용자가 활성화한 QR 수 |
| 실종신고 현황 | 관리대상 상태가 `찾는중`인 수 |
| 광고 진행중 | 광고 상태가 `active`인 수 |
| 월매출 | 선택 월 상품매출 + 구독매출 |
| 상품매출 | 선택 월 결제 완료 상품 단독구매 금액 합계 |
| 구독매출 | 선택 월 결제 완료 구독 주문과 주문 없이 기록된 최신 구독 결제 금액 합계 |

## 구현 파일
- `app/admin/admin-workspace.js`
  - 관리자 메뉴 최상단에 대시보드 추가
  - 보호자 관리 메뉴를 별도 `section=guardians` 경로로 변경
- `app/admin/page.js`
  - 관리자 기본 진입 화면을 대시보드로 변경
  - 월 선택 폼과 8개 지표 카드 추가
  - 보호자 선택/상태 수정 복귀 URL 보정
- `lib/db.js`
  - `getAdminDashboardData()` 추가
  - 회원·대상·QR·실종·광고 건수를 한 번의 집계 쿼리로 조회
  - 한국 시간 기준 선택 월의 상품·구독 매출 조회
  - 주문과 구독 기록이 같은 결제를 가리킬 때 중복 집계 방지
- `app/globals.css`
  - 데스크톱 4열, 중간 화면 2열, 모바일 1열 카드 구성
  - 월매출 카드 강조 및 지표별 색상 구분

## 검증 결과
- Next.js 프로덕션 빌드 성공
- 격리된 임시 SQLite DB에서 다음 기대값과 실제 표시값 일치
  - 전체 회원 2명
  - 관리대상자 3명
  - 활성 QR 1개
  - 실종신고 1건
  - 광고 진행중 1건
  - 상품매출 5,000원
  - 구독매출 36,900원
  - 월매출 41,900원
- 조회 월을 2026년 5월로 바꿨을 때 월매출 7,000원 확인
- 대시보드 기본 진입 및 보호자 관리 메뉴 이동 확인
- 데스크톱 4열, 모바일 1열 및 가로 넘침 없음 확인
- 임시 DB, 테스트 관리자, 화면 캡처 파일 삭제 완료

## 운영 참고사항
- 현재 구독 데이터는 별도 결제 이력 테이블이 아니라 구독 주문과 `subscriptions`의 최신 결제 정보를 함께 사용한다.
- 정기 자동결제가 여러 회차로 확장되면 결제 이력 테이블을 추가해 회차별 매출·취소·환불을 기록하는 것이 필요하다.
- 취소와 환불 매출 차감은 별도 결제 이력 기능이 도입될 때 함께 반영한다.

## 발표자료용 이미지 생성 프롬프트
> 한국 공공서비스 스타일의 관리자 운영 대시보드 화면. 왼쪽에는 접을 수 있는 세로 관리자 메뉴가 있고 대시보드가 파란색으로 활성화되어 있다. 본문에는 전체 회원, 관리대상자, 활성 QR, 실종신고 현황, 광고 진행중, 월매출, 상품매출, 구독매출 총 8개의 카드가 4열로 정돈되어 있다. 상단에는 조회 월 선택기가 있고 월매출 카드는 옅은 파란색으로 강조한다. 흰색 카드, 옅은 회색 배경, 파란색·청록색·초록색·빨간색·노란색의 절제된 상태 색상, 실제 운영 도구처럼 차분하고 전문적인 UI, 한국어 텍스트, 16:9 발표 슬라이드용.
